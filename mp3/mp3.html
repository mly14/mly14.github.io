MP3 Writeup

Overview and Usage

This is a game I created using the P5 and P5 sound JS libraries. The game is pretty straightforward with the objective of keeping a ball in the air with a paddle. If the ball hits the ground - it’s game over. Each time the ball bounces off the paddle, you earn one point. You can click here to play Bounce or you can download the repo here.

Development Process

This is the development process for Bounce.

  1. I took a concept from the previous MP2 with the bouncing ball and turned it into a game. I created a circle that would move until it hit the edge and then it would change directions. I created a rectangle that would act as a paddle from which the ball could bounce off. 
  2. I added a variable to increment the score every time the ball hit the paddle. 
  3. I wanted to add a start screen and an end screen. I took the code from here and modified the styling, and integrated it into my game. To start (and restart) a game, the user can click anywhere in the window.
  4. I added some sound effects for when the ball hits something and also a sound for when the game ends using the P5sound library.

Issue Deep-Dive

One problem I encountered was adding the bounce sound effect. It would play multiple times and sound distorted. I was confused about why that was. After adding print statements, I realized that the draw() function was being called repeatedly which meant that the sound was also being played continuously. After reading the documentation, I learned that draw() “continuously executes the lines of code” unless noLoop() is called. I tried adding that but it didn’t work because I needed to be listening for user clicks since that was how I would be changing the different screens. My solution ended up being really simple. I used an if statement that defaults to true and plays the sound then sets a variable to false so it would only play once. I reset this variable every time the user restarts the game. 

Ideas and Future Work

  • I’d like to use the P5play library because it has a collide function that can make shapes move more realistically. Right now my game is quite predictable and easy so using this new library will make my project more exciting. 
  • Adding an easy, medium, and hard mode could also be interesting. This will involve adding some buttons to the start screen that would change the speed at which the ball moves or the size of the paddle.
